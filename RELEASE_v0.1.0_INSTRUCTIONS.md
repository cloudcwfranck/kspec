## ü§ñ What Happens Automatically

Once you push the tag, the GitHub Actions workflow (`.github/workflows/release.yaml`) will:

1. **Run Tests** - Execute `go test ./...` to ensure everything passes
2. **Build Binaries** - Use GoReleaser to build for all platforms
3. **Generate Checksums** - Create SHA256 checksums for verification
4. **Create GitHub Release** - Publish release with:
   - Comprehensive release notes (from `.goreleaser.yaml`)
   - All binary artifacts
   - Checksums file
   - Automatically generated changelog
5. **Make it Public** - Release will be immediately available

## üêõ If Something Goes Wrong

If the release workflow fails:

1. Check the [GitHub Actions logs](https://github.com/cloudcwfranck/kspec/actions)
2. Common issues:
   - **Tests failing**: Fix tests, commit, create new tag (v0.1.1)
   - **Build errors**: Check Go version compatibility
   - **Permission errors**: Ensure GITHUB_TOKEN has proper permissions

You can always delete and recreate the tag if needed:
```bash
git tag -d v0.1.0
git push origin :refs/tags/v0.1.0
# Fix the issue, then recreate and push
```
